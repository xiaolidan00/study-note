# 字体小于12px

在现代网页开发中，浏览器（尤其是 Chrome）通常会强制将最小字体限制在 **12px**。如果你直接写 `font-size: 10px;`，浏览器往往会无视这个设置，依然显示 12px。

如果你确实需要实现小于 10px 的字体效果，可以尝试以下几种主流方案：

### 1. 使用 CSS `transform: scale()`（最推荐）

这是目前最安全、兼容性最好的做法。先设置一个 12px 的基准字体，然后通过缩放来缩小视觉大小，这样不会触发浏览器的最小字体限制。

```css
.small-text {
  font-size: 12px; /* 满足浏览器最小限制 */
  transform: scale(0.8); /* 视觉上变成 9.6px */
  transform-origin: left center; /* 防止缩放后位置偏移 */
  display: inline-block; /* 确保 transform 生效 */
}
```

### 2. 使用相对单位 `rem` 或 `em`

通过调整根元素或父元素的基准大小，间接让子元素的绝对像素值变小。

```css
html {
  font-size: 8px; /* 将根字体设为 8px */
}
.small-text {
  font-size: 1rem; /* 实际渲染为 8px */
}
```

_注意：部分浏览器（如 Chrome）依然会将 8px 强制渲染为 12px，所以这种方法在某些情况下依然会失效。_

### 3. 使用 SVG 文本

如果是对精度要求极高的图标或特殊排版，可以将文字做成 SVG。SVG 内部的字体大小不受浏览器 CSS 最小字体限制的影响。

```html
<svg width="50" height="15">
  <text x="0" y="12" font-size="8px">极小文字</text>
</svg>
```

### 4. 图片替代

如果这段极小的文字不需要被搜索引擎抓取，也不需要被屏幕阅读器读取（即纯装饰性），直接让设计师切图（PNG/SVG）是最省心的方案。

---

### ⚠️ 体验与无障碍建议

从用户体验（UX）和无障碍设计（Accessibility）的角度来看，**强烈不建议**在网页正文中使用小于 10px 的字体：

1. **可读性差**：在高分屏（Retina）或手机上，极小文字会糊成一团，用户需要凑近屏幕才能看清。
2. **无障碍不达标**：WCAG 标准建议正文最小为 12px-14px，过小的字体会导致视障用户或老年用户无法阅读。
3. **移动端适配**：在手机上，小于 10px 的文字基本等同于不可点击/不可读。

**总结**：如果是必须实现的 UI 细节（如标签角标、极小的辅助说明），请使用 **`transform: scale()`**；如果是正文内容，请尽量将字体保持在 **12px 或以上**。
