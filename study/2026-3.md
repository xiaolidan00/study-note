# ollama AI工具

https://docs.ollama.com/quickstart

使用迅雷下载：`https://github.com/ollama/ollama/releases/latest/download/OllamaSetup.exe`

定位：它是一个开源的推理引擎和包管理器。
作用：它的主要功能是让用户能够极其简单地在本地（Linux, macOS, Windows）下载、安装、运行和管理各种大语言模型。它把复杂的模型部署过程（如配置 GPU、量化、API 接口等）封装成了简单的命令行操作。
开源情况：Ollama 的代码是开源的（主要在 GitHub 上，协议通常为 MIT），你可以查看其源码，甚至贡献代码。
类比：如果把 LLM 比作“电影”，那么 Ollama 就是“播放器”（如 VLC 或 PotPlayer）。播放器是开源的，但它本身不是电影。

```sh
# 安装完后查看版本
ollama --version
# 下载安装模型
ollama run qwen2.5:7b

# 加载模型,选择已经下载的模型，即可启动模型
ollama launch
# 查看已启动的模型
ollama ps

```

## js接入

```ts
import {Ollama} from 'ollama';
async function main() {
  //连接ollama服务
  const ollama = new Ollama({
    host: 'http://10.1.85.11:11434',
    headers: {Authorization: 'Bearer ' + process.env.OLLAMA_API_KEY}
  });
  //查看可用模型
  const list = await ollama.list();
  if (list) {
    console.log('available models:', list);
  }

  //发起聊天
  const question = '请简单回答，为什么天空是蓝色的？';
  console.log('question', question);
  const start = Date.now();
  const response = await ollama.chat({
    //使用模型
    model: 'qwen3.5:2b',
    //聊天消息
    messages: [{role: 'user', content: question}],
    //是否流式
    stream: true
  });

  //流式输出打印结果
  console.log('answer:');
  for await (const part of response) {
    process.stdout.write(part.message.content);
  }
  response.abort();
  console.log('time:', (Date.now() - start) / 1000, 's');
}
main().catch(console.error);
```

```js
import {Ollama} from '@langchain/ollama';

async function main() {
  const llm = new Ollama({
    baseUrl: 'http://10.1.85.11:11434',
    model: 'qwen2.5:latest',
    temperature: 0,
    maxRetries: 2
  });

  const inputText = '32222+44545=?';

  const completion = await llm.invoke(inputText);

  console.log(completion);
}
```

# 有适合本地电脑部署的LLM模型吗

当然有！随着开源模型的快速发展，现在有很多优秀的LLM非常适合在本地电脑上部署。选择的关键在于**平衡模型性能与你的硬件配置（尤其是显存）**。

以下是针对不同硬件档次的推荐方案：

### 一、精选模型推荐（从强到弱排序）

| 模型名称              | 参数量范围      | 最低显存要求 | 核心优势                              | 适合场景                          |
| :-------------------- | :-------------- | :----------- | :------------------------------------ | :-------------------------------- |
| **Llama 3**           | 8B / 70B        | 8GB / 40GB+  | 综合能力顶尖，开源标杆，指令跟随强    | 全能助手，复杂对话与推理          |
| **Qwen 2.5**          | 7B / 32B        | 6GB / 20GB+  | 中文能力突出，数学与代码强，开源友好  | 中文任务，技术问答，数据分析      |
| **Mistral / Mixtral** | 7B / 8x7B/8x22B | 6GB / 20GB+  | 效率高，MoE架构（激活参数少），性能强 | 资源受限下的高性能选择            |
| **Gemma**             | 2B / 7B         | 4GB / 8GB    | 谷歌出品，轻量但能力扎实，安全对齐好  | 入门体验，快速响应，教育用途      |
| **Phi-3**             | 3.8B / 14B      | 4GB / 10GB   | “小钢炮”，体积极小但推理能力惊人      | 超低配置，手机/老旧电脑，边缘设备 |

**参数解释**：

- **最低显存要求**：指运行**量化版本**（如4-bit）的大致需求。若用原版（16-bit），需求会翻数倍。
- **量化**：是本地部署的核心技术，在几乎不损失实用性能的前提下大幅降低内存占用。

### 二、根据你的电脑配置选择

1.  **高性能游戏/工作站**（拥有 **NVIDIA 显卡，显存 ≥ 12GB**，如 RTX 3060/4060 Ti 及以上）
    - **推荐**：**Llama 3 8B**、**Qwen 2.5 32B**、**Mixtral 8x7B** 的 4-bit 量化版。
    - **体验**：可以流畅运行中等规模模型，响应速度快，能力接近初级云端API。

2.  **主流消费级电脑**（拥有 **NVIDIA 显卡，显存 6GB ~ 12GB**，如 GTX 1060/RTX 2060/3050）
    - **推荐**：**Llama 3 8B**、**Qwen 2.5 7B**、**Mistral 7B** 的 4-bit 量化版。
    - **体验**：能运行大多数7B-8B模型，速度尚可，是性价比最高的选择。

3.  **轻薄本/核显电脑**（**无独立显卡或显存很小**，主要依靠 **CPU 和内存**）
    - **推荐**：**Phi-3 mini (3.8B)**、**Gemma 2B**、**Qwen 2.5 1.5B** 等小模型，并使用 **CPU 模式** 运行。
    - **体验**：速度较慢，但能完成基本的问答、摘要、翻译等任务。**大内存（≥16GB）是关键**。

### 三、如何开始部署？（最简单的方法）

无需从零开始，使用以下集成工具可以一键安装和管理：

1.  **Ollama（首选推荐）**
    - **特点**：最简单，跨平台（Win/Mac/Linux），命令行一条指令即可下载和运行模型。
    - **命令示例**：
      ```bash
      ollama run llama3:8b  # 运行 Llama 3 8B
      ollama run qwen2.5:7b # 运行 Qwen 2.5 7B
      ```
    - 它自动处理量化，并提供类ChatGPT的Web界面（通过Open WebUI等）。

2.  **LM Studio**
    - **特点**：漂亮的图形界面，适合不熟悉命令行的用户。内置模型市场，一键下载，并兼容OpenAI API格式，方便其他软件调用。
    - **平台**：Windows/macOS。

3.  **Text Generation WebUI**
    - **特点**：功能最强大、最灵活，支持无数种模型和量化方式，插件丰富。适合高级用户折腾。
    - **平台**：对Windows支持友好。

### 四、部署前的重要提示

- **确认显存**：在任务管理器或 `nvidia-smi` 命令中查看你的**显卡专用显存**，而非共享内存。
- **从“小”开始**：建议先用 **Phi-3** 或 **Gemma 2B** 等小模型测试，成功后再尝试更大的。
- **量化版本是朋友**：优先选择 `Q4_K_M`、`Q5_K_S`、`IQ4_XS` 等后缀的量化模型，能在精度和资源间取得最佳平衡。
- **纯CPU运行**：如果显卡不行，务必在工具设置中选择用CPU运行，并确保系统内存足够（建议16GB以上）。

**总结**：对于绝大多数用户，从 **Ollama** 或 **LM Studio** 开始，尝试 **Llama 3 8B** 或 **Qwen 2.5 7B** 的量化版，是进入本地LLM世界最平滑的路径。
