# 地图适配左右侧栏

```js
onFitView(start, end) {
            this.isAction = true;
            const sideWidth = 340;
            const container = document.getElementById('container');
            const distance = (this.showLeft ? sideWidth : 0) + (this.showRight ? sideWidth : 0);
            const ww = container.offsetWidth - distance;
            const hh = container.offsetHeight;
            const moveCenter = sideWidth * 0.5;
            setTimeout(() => {
                this.isAction = false;
            }, 1000);

            for (let i = 8; i < 18; i++) {
                const p = this.map.project(start, i);
                const p1 = this.map.project(end, i);
                if (Math.abs(p.x - p1.x) >= ww || Math.abs(p.y - p1.y) >= hh) {
                    console.log('fitView', i);
                    const zoom = i - 1;

                    let center = [0.5 * (start[0] + end[0]), 0.5 * (start[1] + end[1])];
                    const p2 = this.map.project(center, zoom);
                    if (this.showLeft && !this.showRight) {
                        center = this.map.unproject([p2.x - moveCenter, p2.y], zoom);
                    } else if (!this.showLeft && this.showRight) {
                        center = this.map.unproject([p2.x + moveCenter, p2.y], zoom);
                    }
                    this.map.setView(center, zoom);
                    console.log('zoom', zoom);
                    this.currentZoom = zoom;
                    return zoom;
                }
            }
        }
```

# DocumentFragment 性能优化

https://developer.mozilla.org/zh-CN/docs/Web/API/DocumentFragment

DocumentFragment 是一个轻量级的文档节点容器。它不在主文档树中，可以看作是一个“虚拟”的 DOM 片段。我们可以在内存中操作这个容器，最后一次性将其内容插入到真实 DOM 中，从而大幅减少浏览器的重排次数。

先说是什么：DocumentFragment 是一个轻量级的文档节点容器，不属于主文档树，用于临时存储 DOM 节点。
再说为什么：直接操作 DOM 会触发重排，频繁操作影响性能。DocumentFragment 在内存中操作 DOM，最后一次性插入，大幅减少重排次数。
然后说怎么用：通过 document.createDocumentFragment()创建，像操作普通 DOM 一样添加子节点，最后一次性插入到目标位置。
最后说应用场景：批量渲染大量数据、动态构建复杂组件、高效移动 DOM 节点等。

# web 端查看 ppt

纯前端解析 PPTX（理论可行，但有限制）
技术点：pptx 是 OpenXML（zip + XML + media）。可以用 JSZip 读 zip，然后解析 XML，渲染成 HTML/SVG。
已知事实：没有成熟、通用且完备的纯前端“pptx→ 原样渲染”库（创建库如 PptxGenJS 用于生成 PPTX，而非渲染）。
优点：不依赖后端或第三方服务；可实现高度自定义渲染。
缺点：需要自己实现布局、SLIDE 转换、字体、动画、复杂样式等，非常耗工且难以兼容所有 PPT 特性。
结论：仅在你接受对兼容性/样式做妥协并愿意开发时考虑此路。
