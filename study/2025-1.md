# vue 与 react 的 diff 算法有什么不同？

相同点：

- 两者的 diff 算法都采用了同级比较，不进行跨级比较，列表都可用 key 快速找到 vnode 复用

不同点：

1. react 的 diff 算法中，采用双缓存机制机制，通过 Fiber 单向链表树遍历对比新旧虚拟 DOM，标记 effectTag 类型，再统一批量更新 DOM。Vue 采用两端到中间的比较方式，并且边对比，边更新 DOM。
2. 节点的 className 改变，vue 认为不是同一个节点，会删除重建来更新，而 react 认为是同一个节点，修改 className 属性。

路径追踪是一种光线追踪，这意味着我们将从每个像素中向世界发射一条光线。对于每条单独的光线，我们将看到它击中的内容，并使用该信息为我们的像素赋予颜色。对每个像素独立执行此操作，即可得到最终图像。

- 进行路径追踪的第一步是计算每个像素的光线。光线有起点和方向，因此我们需要计算光线的起点和方向。

路径追踪在此设置中的工作原理非常简单。我们将像素的颜色从黑色开始，从白色开始“吞吐量”颜色，然后我们将光线射向世界，遵循以下规则：

- 当光线击中对象时，emissive*throughput （自发光*吞吐量） 将添加到像素的颜色中。
- 当光线击中对象时，吞吐量乘以对象的反照率，这会影响未来自发光的颜色。
- 当光线击中物体时，光线将沿随机方向反射，并且光线将继续
- 当光线错过所有对象或达到 N 条光线反弹时，我们将终止。

“吞吐量”当光线从表面反射时，该光线的所有未来照明将乘以该表面的颜色。

uint wang_hash(inout uint seed)
{
    seed = uint(seed ^ uint(61)) ^ uint(seed >> uint(16));
    seed *= uint(9);
    seed = seed ^ (seed >> 4);
    seed *= uint(0x27d4eb2d);
    seed = seed ^ (seed >> 15);
    return seed;
}

float RandomFloat01(inout uint state)
{
    return float(wang_hash(state)) / 4294967296.0;
}

vec3 RandomUnitVector(inout uint state)
{
    float z = RandomFloat01(state) * 2.0f - 1.0f;
    float a = RandomFloat01(state) * c_twopi;
    float r = sqrt(1.0f - z * z);
    float x = r * cos(a);
    float y = r * sin(a);
    return vec3(x, y, z);
}
