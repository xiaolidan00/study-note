# 专业英语

备战软考系统分析师（系分）的专业英语部分，确实需要精准打击。这部分通常占 **5分**（上午综合知识的最后5题），形式是一篇短文填空。

系分的英语考察重点与高项（信息系统项目管理师）不同，它更侧重于 **软件工程、需求分析、系统架构设计** 以及 **新兴技术**。

我为你整理了一份高频核心词汇表，按照考试侧重点进行了分类，助你高效备考。

### 📚 核心高频词汇分类速记

#### 1. 需求工程与分析 (核心考点)

这是系分英语最爱考的领域，特别是关于需求获取和验证的描述。

| 英文术语                                | 中文含义      | 备注/考点提示                                  |
| :-------------------------------------- | :------------ | :--------------------------------------------- |
| **Requirements Validation**             | 需求验证      | 检查需求是否定义了客户真正想要的系统。         |
| **Consistency Checks**                  | 一致性检查    | 确保需求之间没有冲突（如：不应有矛盾的约束）。 |
| **Completeness Checks**                 | 完整性检查    | 确保包含所有功能和约束。                       |
| **Validity Checks**                     | 有效性检查    | 确保系统能执行用户需要的功能。                 |
| **Fact-finding**                        | 事实发现/调查 | 收集需求的技术，如访谈、问卷。                 |
| **Joint Application Development (JAD)** | 联合应用开发  | 一种把用户引入开发过程的事实发现技术。         |
| **Rapid Application Development (RAD)** | 快速应用开发  | 基于团队的技术，加速开发。                     |
| **Use Case**                            | 用例          | 描述系统与参与者交互的场景。                   |
| **Stakeholder**                         | 利益相关者    | 项目涉及的人员（用户、经理、开发人员等）。     |
| **Fishbone Diagram**                    | 鱼骨图        | 用于分析因果关系的工具。                       |

#### 2. 系统架构与设计模式

涉及系统构建方式、设计模式及架构风格。

| 英文术语                                | 中文含义         | 备注/考点提示                                                         |
| :-------------------------------------- | :--------------- | :-------------------------------------------------------------------- |
| **Software Architecture**               | 软件架构         | 系统的基础结构。                                                      |
| **Service-Oriented Architecture (SOA)** | 面向服务的架构   | 常见架构风格，强调服务复用。                                          |
| **Microservices**                       | 微服务           | 将应用构建为一组小型服务。                                            |
| **Model-View-Controller (MVC)**         | 模型-视图-控制器 | 经典的软件设计模式。                                                  |
| **Workflow Reference Model**            | 工作流参考模型   | 包含工作流引擎、过程定义等组件。                                      |
| **Petri Net**                           | Petri网          | 用于描述异步、并发系统的数学建模工具（库所Places、变迁Transitions）。 |
| **Reverse Engineering**                 | 逆向工程         | 读取代码生成模型，用于理解遗留系统。                                  |
| **Prototyping**                         | 原型化           | 快速构建系统模型以获取反馈。                                          |

#### 3. 项目管理与过程改进

虽然系分侧重技术，但项目管理和过程模型也是必考内容。

| 英文术语                                           | 中文含义             | 备注/考点提示                        |
| :------------------------------------------------- | :------------------- | :----------------------------------- |
| **Software Development Life Cycle (SDLC)**         | 软件开发生命周期     | 包含规划、分析、设计、实施等阶段。   |
| **Work Breakdown Structure (WBS)**                 | 工作分解结构         | 将项目目标分解为层次化的活动。       |
| **Critical Path Method (CPM)**                     | 关键路径法           | 用于进度管理。                       |
| **Program Evaluation and Review Technique (PERT)** | 计划评审技术         | 用于分析任务依赖关系，常用于网络图。 |
| **Gantt Chart**                                    | 甘特图               | 展示项目进度的条形图。               |
| **Milestone**                                      | 里程碑               | 标志主要交付产品完成的事件。         |
| **Capability Maturity Model Integration (CMMI)**   | 能力成熟度模型集成   | 过程改进框架。                       |
| **Agile Methodology**                              | 敏捷开发方法论       | 增量式开发，适应变化。               |
| **Scrum / Kanban**                                 | Scrum框架 / 看板方法 | 敏捷开发的具体实践。                 |

#### 4. 新兴技术与基础设施

近年来考试对新技术的考察比例在增加。

| 英文术语                     | 中文含义 | 备注/考点提示               |
| :--------------------------- | :------- | :-------------------------- |
| **Cloud Computing**          | 云计算   | 包括 IaaS, PaaS, SaaS。     |
| **Internet of Things (IoT)** | 物联网   | 万物互联。                  |
| **Big Data**                 | 大数据   | 涉及 Hadoop, MapReduce 等。 |
| **Blockchain**               | 区块链   | 分布式账本技术。            |
| **Containerization**         | 容器化   | 如 Docker, Kubernetes。     |
| **DevOps**                   | 开发运维 | 开发与运维的协作文化。      |
| **Cybersecurity**            | 网络安全 | 涉及加密、防火墙等。        |

### 💡 备考小贴士

1.  **关注上下文逻辑**：系分的英语题通常是完形填空。例如，如果文中提到 "conflict"（冲突）或 "contradictory"（矛盾），答案往往选 **Consistency checks**（一致性检查）；如果提到 "missing functions"（缺失功能），则选 **Completeness checks**（完整性检查）。
2.  **熟悉经典模型**：像 **Waterfall**（瀑布模型）、**Spiral**（螺旋模型）、**Evolutionary**（演化模型）这些词不仅要认识，还要知道它们的优缺点和适用场景。
3.  **积累动词搭配**：注意动词与名词的搭配，例如 "conduct a **preliminary investigation**"（进行初步调查）、"gather **requirements**"（收集需求）。

建议你可以把这些词汇结合历年真题（如2008年、2022年的真题）中的短文进行语境记忆，效果会更好！加油！

# 专业名词缩写

备战软考系统分析师（系分），专业名词缩写是上午综合知识题和下午案例分析题的常客。系分的缩写考察比高项（项目管理师）更侧重于**软件工程、系统架构、遗留系统处理**以及**需求分析**。

我为你整理了系分考试中最高频的专业名词缩写，按知识领域分类，方便你记忆：

### 🏗️ 一、 系统分析与设计（系分核心）

这部分是系分区别于其他考试的特色考点，特别是关于遗留系统和需求分析的工具。

| 缩写     | 英文全称                           | 中文含义             | 考点/备注                                                      |
| :------- | :--------------------------------- | :------------------- | :------------------------------------------------------------- |
| **BSP**  | Business Systems Planning          | **企业系统规划**     | IBM提出，用于大型信息系统规划，核心是定义信息架构。            |
| **DFD**  | Data Flow Diagram                  | **数据流图**         | 结构化分析的核心工具，描述数据在系统中的流动和处理。           |
| **DD**   | Data Dictionary                    | **数据字典**         | 配合DFD使用，对数据流图中所有元素进行定义和说明。              |
| **TFD**  | Transaction Flow Diagram           | **事务流程图**       | 描述业务处理过程和部门分工。                                   |
| **BAM**  | Business Activity Modeling         | **业务活动建模**     | 软系统方法论中的工具，描述业务活动的流动。                     |
| **BPMN** | Business Process Modeling Notation | **业务流程建模符号** | 标准化的业务流程图形化语言。                                   |
| **RE**   | Reverse Engineering                | **逆向工程**         | 分析程序，在比源代码更高的抽象层次上建立系统模型（设计恢复）。 |
| **FE**   | Forward Engineering                | **正向工程**         | 利用恢复的设计信息重构或改善现有系统。                         |
| **EAI**  | Enterprise Application Integration | **企业应用集成**     | 将不同系统连接起来，常考集成模式（如点对点、Hub-and-Spoke）。  |

### 🏛️ 二、 系统架构与新技术

随着考试改革，架构风格和新兴技术的缩写考察频率越来越高。

| 缩写               | 英文全称                                      | 中文含义                     | 考点/备注                                                 |
| :----------------- | :-------------------------------------------- | :--------------------------- | :-------------------------------------------------------- |
| **SOA**            | Service-Oriented Architecture                 | **面向服务的架构**           | 强调松耦合、服务复用，核心组件是**ESB**（企业服务总线）。 |
| **ESB**            | Enterprise Service Bus                        | **企业服务总线**             | SOA的基础设施，用于消息路由、协议转换。                   |
| **Microservices**  | Microservices Architecture                    | **微服务架构**               | 将应用开发为一组小型服务，独立部署。                      |
| **SaaS/PaaS/IaaS** | Software/Platform/Infrastructure as a Service | **软件/平台/基础设施即服务** | 云计算的三种服务模式，必考区分。                          |
| **IoT**            | Internet of Things                            | **物联网**                   | 万物互联，常考架构（感知层、网络层、应用层）。            |
| **Blockchain**     | Blockchain                                    | **区块链**                   | 分布式账本，去中心化，不可篡改。                          |
| **DevOps**         | Development + Operations                      | **开发运维一体化**           | 强调开发、测试、运维的协作与自动化。                      |

### 🛠️ 三、 软件工程与项目管理

这部分很多与高项重叠，但系分更关注开发过程和配置管理。

| 缩写            | 英文全称                                         | 中文含义                            | 考点/备注                                                                   |
| :-------------- | :----------------------------------------------- | :---------------------------------- | :-------------------------------------------------------------------------- |
| **SDLC**        | Software Development Life Cycle                  | **软件开发生命周期**                | 软件从构思到退役的全过程。                                                  |
| **CMMI**        | Capability Maturity Model Integration            | **能力成熟度模型集成**              | 过程改进模型，分5个等级（初始级、已管理级、已定义级、量化管理级、优化级）。 |
| **WBS**         | Work Breakdown Structure                         | **工作分解结构**                    | 将项目范围分解为可管理的任务。                                              |
| **CPM**         | Critical Path Method                             | **关键路径法**                      | 进度管理，决定项目最短工期。                                                |
| **PERT**        | Program Evaluation and Review Technique          | **计划评审技术**                    | 进度管理，使用三点估算处理不确定性。                                        |
| **QA / QC**     | Quality Assurance / Quality Control              | **质量保证 / 质量控制**             | QA侧重过程（预防），QC侧重结果（检查）。                                    |
| **CM**          | Configuration Management                         | **配置管理**                        | 涉及版本控制、基线（Baseline）、变更控制。                                  |
| **MTBF / MTTR** | Mean Time Between Failures / Mean Time To Repair | **平均故障间隔时间 / 平均修复时间** | 系统可靠性指标。可靠性 = MTBF / (MTBF + MTTR)。                             |

### 📊 四、 常用工具与方法论缩写

| 缩写            | 英文全称                                 | 中文含义                            | 考点/备注                            |
| :-------------- | :--------------------------------------- | :---------------------------------- | :----------------------------------- |
| **UML**         | Unified Modeling Language                | **统一建模语言**                    | 包含类图、用例图、时序图、活动图等。 |
| **OOA / OOD**   | Object-Oriented Analysis / Design        | **面向对象分析 / 设计**             | 软件开发的主流方法。                 |
| **MVC**         | Model-View-Controller                    | **模型-视图-控制器**                | 经典的软件架构模式。                 |
| **SQL / NoSQL** | Structured Query Language / Not Only SQL | **结构化查询语言 / 非关系型数据库** | 传统数据库与大数据数据库的区别。     |
| **ETL**         | Extract-Transform-Load                   | **抽取-转换-加载**                  | 数据仓库中的核心数据处理过程。       |
| **OLAP**        | Online Analytical Processing             | **联机分析处理**                    | 数据仓库应用，用于多维数据分析。     |

### 💡 记忆小技巧

1.  **对比记忆**：
    - **QA vs QC**：一个是保证过程不出错，一个是保证产品没瑕疵。
    - **MTBF vs MTTR**：一个是越久越好（耐用），一个是越短越好（好修）。
    - **IaaS vs PaaS vs SaaS**：从底层硬件到上层软件，用户管理的范围越来越小。
2.  **场景联想**：
    - 看到 **“遗留系统”** 就想到 **RE**（逆向工程）和 **FE**（正向工程）。
    - 看到 **“企业集成”** 就想到 **EAI** 和 **ESB**。
    - 看到 **“业务流程”** 就想到 **BPMN** 和 **BSP**。

这些缩写涵盖了系分考试大纲的主要领域，建议结合历年真题中的英语题和案例题进行巩固。祝备考顺利！
